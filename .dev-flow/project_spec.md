# Remote Control 项目规格

## 当前范围：R062 定时终端任务

用户在终端输入框设定时间，到点自动向目标终端发送文本。本质是"延迟发送终端输入"，不是任务编排系统。

### 背景

用户需要"晚点执行"和"每天定时执行"两种场景。Agent 必须在线才能创建定时任务。

### 范围（4 个 Phase，8 个任务）

- **Phase 1** — 后端基础（S001-B002）：数据模型、SQLite 表、CRUD、REST API
- **Phase 2** — 调度引擎（B003）：asyncio 后台协程到点通过 WS DATA 消息发送
- **Phase 3** — 客户端基础（F001-F002）：Dart model、API service、轮询状态、定时标签
- **Phase 4** — 客户端 UI（F003-F004）：长按发送菜单、时间选择器、任务管理

### 产品定义

| 维度 | 决策 |
|------|------|
| 入口 | 输入框 → 长按发送按钮 → 选时间 |
| 形态 | 一次性 + 每日重复，统一走 Server 调度 |
| 前提 | Agent 在线才能创建 |
| 结果 | 不追踪，用户看终端输出 |
| 离线 | 到点 Agent 不在线 → 一次性过期 / 周期跳过 |

## 用户路径

### 桌面端用户

1. 在终端输入框打字
2. 长按发送按钮 → 弹出「5分钟后 / 30分钟后 / 1小时后 / 自定义 / 每日重复」
3. 选择时间 → 输入框清空，终端上方出现 ⏰ 标签
4. 到点 → 终端自动接收文本（一次性标签消失，每日标签保留直到用户取消）

### 移动端用户

1. 同桌面端流程
2. 长按发送按钮触发 BottomSheet 选择时间

## 目标平台

- Server: Docker (Linux x86_64)
- Client: macOS arm64 + Android
- Agent: 不改动
