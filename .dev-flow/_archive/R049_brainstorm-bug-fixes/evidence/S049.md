# S049: Fix tool registration name parameter

## 问题

Pydantic AI `agent.tool()` 默认使用函数 `__name__` 作为工具名。内部工具函数以 `_` 前缀命名（如 `_tool_execute_command`），GLM-5.1 将其与调用拼接产生无效工具名 `_tool_execute_command_tool_execute_command`。

## 修复

为所有内置工具注册添加显式 `name` 参数：

### terminal_agent_tools.py

`_register_builtin_tools()`:
```python
agent.tool(_tool_execute_command, name="execute_command")
agent.tool(_tool_ask_user, name="ask_user")
agent.tool(_tool_deliver_result, name="deliver_result")
```

`_register_lookup_knowledge()`:
```python
agent.tool(_tool_lookup_knowledge, name="lookup_knowledge")
```

### terminal_agent.py

全局注册同步更新：
```python
terminal_agent.tool(_tool_execute_command, name="execute_command")
terminal_agent.tool(_tool_ask_user, name="ask_user")
terminal_agent.tool(_tool_lookup_knowledge, name="lookup_knowledge")
terminal_agent.tool(_tool_deliver_result, name="deliver_result")
```

## 测试更新

`tests/test_terminal_agent.py` 中 `TestSessionAgentFactory` 断言从旧名称（`_tool_execute_command` 等）更新为新名称（`execute_command` 等）。
