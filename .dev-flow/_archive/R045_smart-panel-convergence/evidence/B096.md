## Task Snapshot

- **Phase**: eval-core
- **Module**: server
- **Priority**: 1
- **Verification**: L1
- **Risk Tags**: config
- **Started At**: 2026-04-25

### Acceptance Criteria

1. server/evals/__init__.py, models.py, db.py 创建
2. EvalTaskDef 支持 YAML 加载（id/category/description/input/expected/graders/metadata）
3. EvalTrial 记录 transcript + duration + token_usage
4. EvalGraderResult 记录 grader_type + score + details
5. evals.db 6 张表创建成功（eval_task_defs / eval_trials / eval_grader_results / eval_runs / quality_metrics / eval_task_candidates）
6. EVAL_AGENT_MODEL/BASE_URL/API_KEY 缺失时 raise 明确错误，不复用 ASSISTANT_LLM_* 变量
7. quality_metrics 表：按 session_id 存储质量指标
8. eval_task_candidates 表：存储反馈闭环的 candidate task

### Contract Ref

- CONTRACT-051 (api_contracts.md)

### Architecture Constraints

- eval 框架在 server/evals/ 独立包内，不侵入业务代码
- eval 使用独立 evals.db，与主业务 app.db 隔离
- 评估体系所有模型配置独立于业务 agent，不复用 ASSISTANT_LLM_* 变量
- EVAL_AGENT_MODEL + EVAL_AGENT_BASE_URL + EVAL_AGENT_API_KEY 必填
- 不允许任何默认模型回退
- eval 不写 app.db，只写 evals.db
