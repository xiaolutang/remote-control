# S505: Server verify_token/verify_refresh_token 重复代码消除

## 状态: PASS

## 变更摘要

提取 `verify_token` 和 `verify_refresh_token` 中约 70% 的重复逻辑为两个公共辅助函数：

### 新增辅助函数

1. **`_validate_token_format(token, token_label)`** (行 134-160)
   - 空值检查 (401)
   - 长度检查 > 10KB (413)
   - null bytes 检查 (400)
   - 参数化 `token_label` 用于差异化错误提示

2. **`_decode_and_classify_jwt(token, token_label)`** (行 163-196)
   - JWT 解码
   - 错误分类：expired -> TOKEN_EXPIRED，其他 -> TOKEN_INVALID
   - 返回 `TokenVerificationError`（继承 HTTPException）

### 重构后函数

- **`verify_token`**: 80 行 -> 26 行（不含 docstring），调用两个辅助函数
- **`verify_refresh_token`**: 75 行 -> 32 行，调用两个辅助函数
  - 捕获 `TokenVerificationError` 转换为 `HTTPException`，保持对外行为不变（不带 error_code）

### 行为保持

- `verify_token` 的错误类型 `TokenVerificationError` 不变
- `verify_refresh_token` 的错误类型 `HTTPException` 不变
- 所有 status_code、detail 文本、返回值结构不变
- 函数签名不变（公共 API 兼容）
