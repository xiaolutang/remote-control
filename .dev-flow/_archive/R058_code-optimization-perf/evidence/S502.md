## 问题分析

`agent/app/core/env_compat.py` 第54行运算符优先级 bug：

```python
# 修复前 (BUG)
if path and "/opt/homebrew" in path or "/usr/local" in path:
```

Python 运算符优先级：`and` 高于 `or`，等价于：
```python
if (path and "/opt/homebrew" in path) or ("/usr/local" in path):
```

当 `path` 为空字符串 `""` 时：
- `(path and "/opt/homebrew" in path)` → `"" and ...` → `""` (falsy)
- `"/usr/local" in path` → `False`（不会抛异常，因为空字符串也支持 `in` 操作符）

但当 `path` 为 `None` 时（理论上 strip 后不会，但防御性编程仍需要）：
- `"/usr/local" in None` → `TypeError`

## 修复方案

```python
# 修复后
if path and ("/opt/homebrew" in path or "/usr/local" in path):
```

添加括号明确优先级：先检查 `path` 非 empty/falsy，再检查内容。
