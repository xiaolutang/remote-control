## 变更摘要

提取 `_request_agent_with_ack()` 通用函数，消除了 `request_agent_create_terminal()` 和 `request_agent_close_terminal_with_ack()` 之间约 80% 的代码重复。

## 修改文件

- `server/app/ws/agent_request.py`

## 重构细节

### 提取的通用函数: `_request_agent_with_ack()`

参数化以下差异点:
- `registry_name`: "terminal_creates" / "terminal_closes"
- `future_key`: (session_id, terminal_id) 两种场景相同结构
- `message`: 发送给 agent 的消息内容
- `timeout`: 超时秒数
- `conflict_detail`: 409 冲突提示文案
- `error_label`: "创建" / "关闭"，用于构造错误信息

封装的公共逻辑:
1. `get_agent_connection()` — 不在线抛 409
2. `pending_registry.create_tuple_future()` — 创建 pending future
3. `agent_conn.send()` — 发送消息
4. `asyncio.wait_for()` — 等待确认
5. RuntimeError 处理 — agent disconnected / offline → 409, 其他 → 500
6. TimeoutError 处理 — → 504
7. 异常时 `pop_tuple_future()` 清理

### 重构后的函数

- `request_agent_create_terminal()`: 调用 `_request_agent_with_ack()` 后，额外查询 session terminal 并返回
- `request_agent_close_terminal_with_ack()`: 调用 `_request_agent_with_ack()` 后，返回 `{"terminal_id", "reason"}`
- `request_agent_close_terminal()`: 未改动（best-effort，不需要 ack）

### 保持不变

- 函数签名完全不变
- 返回值格式不变
- 外部行为不变
