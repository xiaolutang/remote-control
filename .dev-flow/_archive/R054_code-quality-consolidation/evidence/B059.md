# B059: 效率优化 -- 心跳 + 连接去重 + history

## 变更摘要

| 文件 | 改动 | 说明 |

| `server/app/ws/agent_connection.py` | +2 行 | AgentConnection 新增 `_redis_agent_online` 缓存字段 |

| `server/tests/test_session.py` | +282 行 | 9 个新测试覆盖三个优化点 |

## 优化 1: 心跳

**Before**: 每 30s 心跳无条件调用 `update_session_device_heartbeat`（完整 Redis 读写：GET session JSON -> 修改 -> SET）
**After**: 首次心跳调用 `get_session` 读取 `agent_online` 状态并缓存到 `AgentConnection._redis_agent_online`；后续心跳如缓存为 True，直接跳过 Redis 写入

## 优化 2: 连接去重

**Before**: Client 连接流程调用 `get_session_terminal`（独立的 Redis 读取+锁）验证 terminal 存在性
**After**: 直接从已获取的 `session` dict 中的 `terminals` 列表查找，零额外 Redis 开销

## 优化 3: history 渐进式读取

**Before**: 一次读取 `min(total, 5000)` 条，内存中过滤
**After**: 有过滤条件时，首次读取 500 条，匹配数不足 `offset + limit` 时扩大到 1500→5000；无过滤条件时直接 5000
