## 实现设计

**缓存策略**: 在 `_build_agent_conversation_projection` 中，仅对无 `after_index` 的全量投影路径添加 TTL 缓存。

- **缓存 key**: `{user_id}:{device_id}:{terminal_id}`（唯一标识一个 terminal 的 conversation）
- **TTL**: 500ms（`_PROJECTION_CACHE_TTL = 0.5`）
- **存储**: 模块级 dict `_projection_cache`，值 `(monotonic_timestamp, deep_copy_of_projection)`
- **命中**: `after_index is None` + 缓存 key 存在 + TTL 未过期 → 返回 deep copy
- **写入**: 仅 `after_index is None` 且 conversation 存在且 active 时写入
- **失效**: `_publish_conversation_stream_event` 中调用 `_invalidate_projection_cache`，新事件到达时立即清除

**增量路径跳过缓存**: 有 `after_index` 的调用完全跳过缓存读写，直接查询 SQLite。

**数据安全**: 缓存写入时 deep copy，命中时也返回 deep copy，确保多次调用互不影响。
