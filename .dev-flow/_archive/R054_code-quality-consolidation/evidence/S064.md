# S064: Agent 清理 — alias shim + _log 提取 + CONFIG_DIR 统一

## 变更摘要

### 1. Alias shim 清理（10 个文件删除）
删除以下 alias shim 文件，所有 import 已迁移到新路径：

| shim 文件 | 重导出目标 |
|-----------|-----------|
| `app/config.py` | `app.core.config` |
| `app/auth_service.py` | `app.security.auth_service` |
| `app/crypto.py` | `app.security.crypto` |
| `app/command_validator.py` | `app.security.command_validator` |
| `app/log_adapter.py` | `app.core.log_adapter` |
| `app/pty_wrapper.py` | `app.core.pty_wrapper` |
| `app/websocket_client.py` | `app.transport.websocket_client` |
| `app/knowledge_tool.py` | `app.tools.knowledge_tool` |
| `app/skill_registry.py` | `app.tools.skill_registry` |
| `app/mcp_client.py` | `app.tools.mcp_client` |

### 2. _log() 函数统一
- `core/log_adapter.py` 新增 `_log()` 函数（唯一定义处）
- `transport/websocket_client.py` 删除本地 `_log()`，改为从 `core/log_adapter` 导入
- `transport/agent_message_handler.py` 删除本地 `_log()`，改为从 `core/log_adapter` 导入
- `cli.py` 的 `_log_agent` 改为从 `core/log_adapter` 导入 `_log as _log_agent`

### 3. DEFAULT_CONFIG_DIR 统一
- `core/config.py` 新增 `DEFAULT_CONFIG_DIR = os.path.expanduser("~/.rc-agent")`
- `core/config.py` 的 `get_config_path()` 引用 `DEFAULT_CONFIG_DIR`
- `security/crypto.py` 引用 `DEFAULT_CONFIG_DIR` 替代 `Path.home() / ".rc-agent"`
- `tools/knowledge_tool.py` 引用 `DEFAULT_CONFIG_DIR` 替代硬编码 `"~/.rc-agent"`
- `tools/skill_registry.py` 引用 `DEFAULT_CONFIG_DIR` 替代硬编码 `"~/.rc-agent"`

### 4. rc-agent.spec 更新
- 移除旧路径的 hiddenimports（`app.config`, `app.auth_service`, `app.crypto` 等）
- 添加新路径的 hiddenimports（`app.core.*`, `app.security.*`, `app.transport.*` 等）
