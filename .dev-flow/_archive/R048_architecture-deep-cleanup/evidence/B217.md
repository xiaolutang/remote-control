# B217: eval integration 自动启动 Agent 进程

## Execution Evidence

### Files Changed
- `server/evals/integration.py`
- `server/evals/__main__.py`
- `server/tests/test_eval_integration.py`

### Implementation
- `IntegrationRunner` 新增：
  - `_compose_cmd()` 统一生成 `docker compose` 命令
  - `_load_env_values()` / `_require_agent_credentials()` 在启动前校验 `AGENT_USERNAME`、`AGENT_PASSWORD`
  - `start_agent()` 通过 `docker compose run --rm --use-aliases agent` 启动独立 Agent 进程
  - `wait_agent_online()` 轮询 `eval_client.get_or_create_device()`，并在 Agent 提前退出时快速失败
  - `stop_agent()` 负责终止 compose run 进程并配合 `tear_down()` 清理
- `evals.__main__` 的 integration 执行链调整为：
  - `build_and_deploy()`
  - `wait_for_healthy()`
  - `start_agent()`
  - `eval_client.setup()`
  - `wait_agent_online()`
  - `run tasks`
  - `stop_agent()`
  - `tear_down()`

### Covered Scenarios
- 缺少 `AGENT_USERNAME/AGENT_PASSWORD` 时快速失败
- `start_agent()` 正确发起 `docker compose run`
- `wait_agent_online()` 轮询成功返回 `device_id`
- Agent 提前退出时 `wait_agent_online()` 优雅报错
- `stop_agent()` 清理 compose run 进程
- CLI integration 路径显式调用 `start_agent / wait_agent_online / stop_agent`
