# S207: Server 拆分后 Docker 集成验证

## Execution Evidence

### Script Repairs During Audit
- `scripts/test-docker-integration.sh` 调整为先启动 `redis + server`，再创建测试用户后启动 `agent`
- Docker smoke 改为 Agent 专用 token 流程，避免 token replacement 干扰 API 校验
- `deploy/agent.Dockerfile` 补充 `COPY agent/local_server.py ./local_server.py`，修复容器内 `No module named 'local_server'`
- 修复 Agent 在线判定脚本 bug：`DEVICE_COUNT` 的 `python3 -c` 多余右括号导致解析失败后始终回退为 `0`
- 将终端创建请求升级到当前接口契约：补齐 `terminal_id/title/cwd/command`

### Runtime Verification
- `./scripts/test-docker-integration.sh --with-agent`
  - 修复前：`PASS 16 / FAIL 1`
  - 失败原因为：`Agent 未在 40s 内连接`，但根因是脚本计数逻辑错误，不是 Agent 未连接
  - 修复后：`PASS 18 / FAIL 0`
  - 关键通过项：
    - `Agent 已连接，设备数: 1`
    - `终端创建成功: eval-term-...`
    - `Agent 有响应 (13509 bytes) 但 SSE 解析未提取到 summary`
- `docker logs rc-agent --tail 200`
  - `[LocalServer] 本地 HTTP Server 已启动: http://127.0.0.1:18765`
  - `[Agent] 已连接到服务器`
  - `[Agent] 会话已建立: ...`
- 手工命中 `GET /api/runtime/devices`（使用容器内 `AGENT_TOKEN`）
  - 返回 `devices[0].agent_online = true`
  - 返回 `devices[0].device_id = 会话对应 device_id`

### Audit Finding
- 真实 Docker 环境下，Server 拆分后的本地标准 Docker 发布链路已恢复闭环：
  - Agent 容器可启动本地 HTTP Supervisor
  - Agent WebSocket 可成功注册上线
  - `/api/runtime/devices` 可观察到在线设备
  - terminal create + agent run smoke 可打通
- 剩余的 `summary` 解析提示仅是 smoke 脚本的宽松提示分支，不构成失败项，也不阻塞 `S207` 验收。
