# B108: Eval harness 对齐 deliver_result 工具 + grader 兼容

## 任务概述

更新 eval harness 以支持 deliver_result 工具新架构：
1. harness.py 添加 deliver_result + ask_user + lookup_knowledge 为可调用工具
2. LLM 调用 deliver_result 时捕获参数并校验
3. 完成判定从"无工具调用+文本JSON"改为"deliver_result 被调用"
4. tool_call_order grader 排除 deliver_result
5. eval system prompt 对齐生产 SYSTEM_PROMPT
6. LLM 未调用 deliver_result 时 trial 标记为 incomplete
7. deliver_result 参数校验镜像生产 AgentResult 约束

## 修改文件清单

### `server/evals/harness.py`
- `_validate_deliver_result()`: 新增函数，校验 deliver_result 参数（镜像生产 AgentResult 约束）
  - command: steps 非空、无 ai_prompt
  - ai_prompt: ai_prompt 非空、need_confirm=True、steps 空
  - message: steps 空、无 ai_prompt、need_confirm=False
- `_build_tools_schema()`: 返回 4 个工具定义（execute_command + ask_user + lookup_knowledge + deliver_result）
  - deliver_result schema 包含完整字段：response_type/summary/steps/ai_prompt/provider/source/need_confirm/aliases
- `_build_system_prompt()`: 延迟导入生产 SYSTEM_PROMPT
- `_execute_single_trial()`:
  - deliver_result 调用时校验参数，无效参数标记 error 结果
  - deliver_result 后立即 break（不处理同 turn 后续工具）
  - ask_user 返回 mock 回复
  - lookup_knowledge 返回降级回复
  - max_turns 用尽但无 deliver_result 标记 incomplete
  - 无工具调用也标记 incomplete
  - incomplete trial 的 agent_result 设为 error 类型

### `server/evals/graders/code_grader.py`
- `ToolCallOrderGrader`: EXCLUDED_TOOLS = {"deliver_result"}
- `StepsStructureGrader`: 更新为 response_type 感知
  - message: 校验 steps 空 + 无 ai_prompt + need_confirm=False
  - ai_prompt: 校验 steps 空 + ai_prompt 非空
  - command: 校验 steps 非空

### `server/tests/test_eval_harness.py`
- `TestDeliverResultTool`: 7 个核心测试
- `TestDeliverResultValidation`: 7 个校验测试（command 空 steps、ai_prompt 空 prompt、message 带 steps、message need_confirm、message ai_prompt、command ai_prompt、deliver_result 终止后续工具）
- 更新 9 个旧测试的 mock LLM 响应格式

### `server/tests/test_eval_graders.py`
- `TestToolCallOrderGrader`: 3 个 deliver_result 排除测试
- `TestStepsStructureGrader`: 5 个新增 response_type 感知测试
