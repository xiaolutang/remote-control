### Acceptance Criteria

1. AppEnvironment 枚举包含 local 和 production
2. EnvironmentService 只管理环境状态，无任何副作用调用
3. 选择 local 时生成 ws://{host}:{port}/rc
4. 选择 production 时生成 wss://xiaolutang.top/rc
5. Debug 模式默认 local，Release 默认 production（通过可注入 provider）
6. 本地 host/port 修改后持久化到 SharedPreferences
7. AppConfig.toJson() 不包含 serverUrl 字段
8. AppConfig.fromJson() 使用传入的 serverUrl 而非 json 中存储的值
9. ConfigService.loadConfig() 使用 EnvironmentService 的 serverUrl
10. EnvironmentService 不依赖 AuthService、DesktopAgentManager 或任何 UI 组件

### Architecture Constraints

- 不变量 #25: serverUrl 的单一真相源是 EnvironmentService，AppConfig 不持久化 serverUrl
- 不变量 #26: 环境切换编排由 UI/协调层触发，EnvironmentService 不做任何副作用
- 禁止: EnvironmentService 直接调用 AuthService.logout / DesktopAgentManager
