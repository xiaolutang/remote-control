## Acceptance Criteria

1. POST /api/feedback 接收 category + description + platform + app_version，返回 feedback_id
2. Server 自动从 Redis log_service 查询该 session_id 最近 50 条日志并关联
3. 反馈存储到 Redis Hash rc:feedback:{id}，列表存储到 rc:user_feedbacks:{user_id}
4. 使用 async_verify_token 鉴权，token 无效返回 401
5. 转发到 log-service 是 best-effort，不阻塞响应
6. category 限定枚举：connection / terminal / crash / suggestion / other

## Architecture Constraints

- 所有受保护路由必须使用 async_verify_token（不变量 #18）
- TokenVerificationError 必须透传到 HTTP 响应层（不变量 #19）
- Redis 不可用时 fail-closed（不变量 #17）
