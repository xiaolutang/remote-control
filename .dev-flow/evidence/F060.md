### Acceptance Criteria

1. LoginScreen 无 serverUrl 构造参数，从 EnvironmentService 读取
2. 登录页顶部有环境选择控件（本地/线上）
3. 选择本地时显示 host + port 编辑输入框
4. 选择线上时不显示编辑框
5. 切换环境时：停 Agent → 断终端 → 清凭证 → 更新环境 → 刷新 UI
6. 登录/注册使用 EnvironmentService.currentServerUrl
7. 环境选择持久化，下次打开保持
8. SplashPage 使用 EnvironmentService.currentServerUrl 做自动登录
9. SplashPage 两处 LoginScreen 构造均为无参
10. logout_helper 不再接受 serverUrl 参数，改从 EnvironmentService 读取
11. terminal_screen.dart 两处 token 失效回登录页均为无参 LoginScreen
12. TerminalWorkspaceScreen/UserProfileScreen 登出后跳转无参 LoginScreen
13. 桌面端 Agent 配置使用正确环境的 serverUrl
14. EnvironmentService 不调用任何 AuthService/AgentManager 方法
15. Debug 模式默认选中本地环境
16. Release 模式默认选中线上环境

### Architecture Constraints

- 不变量 #25: serverUrl 单一真相源是 EnvironmentService
- 不变量 #26: 环境切换编排由 UI/协调层触发
- 禁止: LoginScreen 接受 serverUrl 构造参数
