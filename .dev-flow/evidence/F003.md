## Design Decisions

### 离线路径
移动端设备离线时，DesktopWorkspaceController._deriveWorkspaceState 返回 deviceOffline，
_buildBody 在此状态下返回 WorkspaceEmptyState 而非渲染 CompactTabStrip。
这是正确的架构决策：终端连接已断开，Tab 栏无意义。
离线场景的"创建失败"由 deviceOffline 空态页面的"刷新状态"按钮处理。

### 创建可用性与错误处理
createDisabled 统一为 `!device.canCreateTerminal || controller.creatingTerminal`：
- 设备离线：canCreateTerminal=false，但此时 CompactTabStrip 不渲染
- 达到上限：canCreateTerminal=false，按钮禁用（与桌面端一致）
- 创建中：creatingTerminal=true，按钮禁用（与桌面端一致）
- 创建 API 失败：移动端通过 _createEmptyTerminal(snackBarOnError: true) 用 SnackBar 局部提示

### 移动端创建失败统一处理
所有移动端创建入口统一使用 `snackBarOnError: !controller.isDesktopPlatform`：
- workspace-empty-create-action（空状态创建按钮）
- workspace-menu-create（菜单创建）
- compact-tab-create（CompactTabStrip + 按钮）
桌面端不受影响（snackBarOnError=false）。

### Slot 位置
CompactTabStrip 通过 bottomChrome slot 插入 TerminalScreen 的 Column，
位于 terminal-touch-layer 下方、TerminalShortcutBar 上方（测试 14 验证）。
