## Fix 1: 循环依赖解耦

**问题**: `services/agent_session_manager.py` 通过延迟导入 `from app.api.runtime_api import _publish_conversation_stream_event`，违反分层原则（services 不应依赖 api）。

**方案**: 创建独立的 pub/sub 模块 `infra/event_bus.py`，提供 `publish_conversation_stream_event` / `subscribe_conversation_stream_event` / `unsubscribe_conversation_stream_event` 三个接口。

**修改文件**:
- **新增** `server/app/infra/event_bus.py` — 轻量级事件总线，维护 handler 注册表
- **修改** `server/app/services/agent_session_manager.py` — 顶部导入 `from app.infra.event_bus import publish_conversation_stream_event`，替换函数内的延迟导入
- **修改** `server/app/api/agent_conversation_helpers.py` — 导入 event_bus，`_publish_conversation_stream_event` 同时通知本地 SSE queue + event_bus
- **修改** `server/app/api/runtime_api.py` — 增加 `publish_conversation_stream_event` 重导出

**验证**: `grep -n "from app.api" app/services/agent_session_manager.py` 返回空

## Fix 2: 管道符拦截

**问题**: `_SHELL_META` 正则缺少 `|`，`ls | grep secret` 可通过验证。

**方案**: 正则从 `[;&$\`\\]|>>|>` 改为 `[;|&$\`\\]|>>|>`

**修改文件**:
- `server/app/infra/command_validator.py`
- `agent/app/command_validator.py`（同步更新 Agent 端副本）

**验证**:
- `test_command_validator.py::TestValidateCommandShellMeta::test_pipe` PASSED
- `test_terminal_agent.py::TestSecurityAttackVectors::test_shell_meta_commands_blocked[ls | grep secret]` PASSED

## Fix 3: command 字段

**问题**: `_tool_step_event()` 函数构建的 payload 不包含 `command` 字段，导致 `_extract_tool_args_from_event` 需要反向解析 description 来获取命令。

**方案**: `_tool_step_event()` 新增 `command` 关键字参数（默认空字符串），execute_command 回调调用处传入 `command=command`。

**修改文件**:
- `server/app/services/agent_session_manager.py`

**验证**: `_tool_step_event('execute_command', 'test', 'running', command='ls -la')` 返回包含 `'command': 'ls -la'`
