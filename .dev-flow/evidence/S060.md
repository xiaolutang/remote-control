## Key Implementation Details

### --config → RC_AGENT_CONFIG_DIR Bridge
1. `RC_AGENT_CONFIG_DIR` env var is set to config file's parent directory (resolved to absolute path)
2. This takes precedence over any pre-existing env var
3. `skill_registry.py`, `knowledge_tool.py`, and `crypto.py` all use `RC_AGENT_CONFIG_DIR` as their data root
4. `crypto.py` uses lazy property to defer state dir resolution until after env var is set

### Strict Config Loading
- When `--config` is explicit, `load_config(strict=True)`:
  - File exists but unreadable: OSError → Error exit 1
  - File contains malformed JSON: JSONDecodeError → Error exit 1
  - File does not exist: Warning, defaults used (expected first-run behavior)

### Hidden Dependencies Handling
- `log-service-sdk`: collect_submodules + collect_data_files
- `cryptography`: collect_submodules + collect_data_files (native .so)
- `local_server`: hiddenimports
