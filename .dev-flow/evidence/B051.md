## Acceptance Criteria
1. 同一 terminal 连续两次 agent run，第二次复用同一个 session_id（不创建新 session）
2. 不同 terminal 的 agent run 使用不同 session_id
3. 终端删除时关联的 agent session 被清理
4. 终端因设备离线、用户登出、认证失效等非删除原因关闭时，session 标记为 inactive
5. 同一终端重连或下次 run 时，inactive session 自动恢复为 active 并继续复用
6. agent 内部 run 概念保留
7. SSE 事件中 session_created 只在首次 run 时发送
8. usage_store 同一 session_id 多次累加正确
9. Redis 不可用时 session 查找返回 503
10. 首次 agent run LLM 失败时不创建 session 记录
11. 同一 terminal 多次 run 的 execution report 按 run_id 区分
12. GET /api/agent/usage/summary?terminal_id=yyy 返回 per-terminal 累计 usage
13. terminal 无 session 时 usage summary 返回零值
14. terminal_id 不属于该 device_id 时返回空零值
15. pytest 全部通过

## Execution Evidence

### Implementation (2026-04-29)

**核心改动**：

1. **agent_session_types.py** — 新增 `INACTIVE` 状态（终端非删除关闭）
2. **agent_session.py** — 新增 `run_count`, `is_first_run`, `current_run_id` 字段
3. **agent_session_manager.py** — 新增方法：
   - `generate_terminal_session_id(terminal_id)` — 确定性 session_id 派生
   - `get_terminal_session(user_id, device_id, terminal_id)` — 按 terminal 查找任意状态 session
   - `reuse_or_create_session(...)` — 复用/创建 per-terminal session
   - `remove_terminal_sessions(device_id, terminal_id)` — 终端删除清理
   - `mark_session_inactive(session_id)` — 标记为 inactive
4. **agent_api.py** — `run_terminal_agent_session` 改用 `generate_terminal_session_id(terminal_id)` 替代随机 uuid4
5. **usage_store.py** — `save_agent_usage` 新增 `terminal_id` 参数；`get_usage_summary` 支持 `terminal_id` 过滤
6. **database.py** — 代理函数同步更新签名
7. **agent_usage_api.py** — `get_agent_usage_summary_api` 新增 `terminal_id` 查询参数
8. **_schema.py** — 新增迁移 `ALTER TABLE agent_usage_records ADD COLUMN terminal_id TEXT DEFAULT ''`

**不变量**：
- #64: session 与 terminal 是 1:1 关系，lazy creation on first run
- #65: 服务器是 token usage 唯一权威来源
- 禁止创建新 agent session_id per question
- 禁止客户端自行累加 token usage
