## 实现细节

### SQL 累加逻辑
- `ON CONFLICT(session_id) DO UPDATE` 改为累加模式：
  - `input_tokens = COALESCE(agent_usage_records.input_tokens, 0) + excluded.input_tokens`
  - `output_tokens` / `total_tokens` / `requests` 同理
- `model_name` 使用 CASE 表达式保留最新非空值
- `created_at` 更新为最新时间

### 负数防护
- 在 `save_agent_usage()` 入口处对所有 token/request 值做 `max(0, int(value or 0))` 防护

### COALESCE 防御
- 累加 SQL 使用 `COALESCE(existing, 0) + excluded` 防御 NULL 边界情况
