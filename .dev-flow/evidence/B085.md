## Execution Evidence

Implemented storage-layer support for terminal-bound Agent conversations:

- Added `agent_conversations` table with `conversation_id`, `user_id`, `device_id`, `terminal_id`, `status`, `tombstone_until`, timestamps, and unique `user_id + device_id + terminal_id`.
- Added `agent_conversation_events` table with `event_index`, `event_id`, `event_type`, `role`, `session_id`, `question_id`, `client_event_id`, payload JSON, and cascade delete.
- Added indexes for conversation scope, event ordering, idempotent `client_event_id`, and single-answer `question_id`.
- Added repository methods:
  - `get_or_create_agent_conversation`
  - `get_agent_conversation`
  - `append_agent_conversation_event`
  - `list_agent_conversation_events`
  - `close_agent_conversation`
  - `delete_agent_conversation`
  - `cleanup_agent_conversation_tombstones`
- Added `AgentConversationConflict` with `question_already_answered` and `closed_terminal` conflict codes.
- Made event append use `BEGIN IMMEDIATE` before assigning `event_index`.
- Made close use a single transaction for `closed` event append and `closed` tombstone marking.
- Made first conversation creation use `INSERT OR IGNORE` to avoid concurrent create unique-key failures.

## Acceptance Check

- Same `user_id + device_id + terminal_id` reuses one conversation.
- Different terminal IDs get isolated conversation IDs.
- Events receive monotonic `event_index` and support `after_index`.
- Same `client_event_id` replay returns original event without appending.
- A second answer for the same `question_id` raises `question_already_answered`.
- Closed tombstone hides historical events and cleanup physically deletes the conversation.
- Physical delete cascades events.
- Cross-user reads return no conversation/events.
