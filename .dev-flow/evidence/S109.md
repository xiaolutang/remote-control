## 迭代修复

### Fix 1: need_confirm 从 response_type 推断（Harness）
- **文件**: `server/evals/harness.py`
- **改动**: 当模型未显式设置 need_confirm 时，从 response_type 推断默认值
  - message → False, command → True, ai_prompt → True
- **效果**: 立即消除 10 个 validation error

### Fix 2: response_type 选择规则明确化（SYSTEM_PROMPT）
- **文件**: `server/app/terminal_agent.py`
- **改动**: 重写 response_type 选择规则，明确三种类型的使用场景
  - command: 直接终端操作、启动工具、简单命令
  - ai_prompt: 仅用于复杂编程任务需要 AI 辅助
  - message: 知识问答、超出范围

### Fix 3: 强制调用 deliver_result（SYSTEM_PROMPT）
- **文件**: `server/app/terminal_agent.py`
- **改动**: 将"结果交付方式"改为"结果交付方式（强制）"，明确所有场景都必须调用 deliver_result
  - 添加具体示例（问候→message、问答→message、危险→message、编程→command/ai_prompt）

### Fix 4: 区分 execute_command 限制和 deliver_result 能力（SYSTEM_PROMPT）
- **文件**: `server/app/terminal_agent.py`
- **改动**: 明确 execute_command 只能只读探索，但 deliver_result(response_type='command') 可以建议任何命令
  - 只有安全边界外的请求才用 message 拒绝
