# S510: Agent 硬编码常量集中到 Config

## 变更摘要

将 Agent 模块中 16+ 处散落的硬编码值分为两类集中管理：
1. **运行时可配值** → 添加到 `Config` 模型（可通过 JSON/CLI 配置）
2. **内部代码常量** → 定义为模块级 `_UPPER_CASE` 常量（不暴露给用户）

## 修改文件

### 1. `agent/app/core/config.py` — 新增运行时可配属性
- `retry_delay: float = 1.0` — 重连基础延迟（秒），指数退避基数
- `heartbeat_interval: int = 30` — 心跳间隔（秒）

### 2. `agent/app/transport/websocket_client.py` — 模块级内部常量
| 常量 | 值 | 用途 |
|------|-----|------|
| `_MAX_RETRY_DELAY` | 60.0 | 指数退避上限 |
| `_AUTH_RECV_TIMEOUT` | 30 | auth 响应等待超时 |
| `_RECV_POLL_TIMEOUT` | 1 | WebSocket recv 轮询超时 |
| `_STDIN_BUF_SIZE` | 1024 | 本地 stdin 缓冲区 |
| `_STDIN_READ_TIMEOUT` | 0.1 | stdin 读取超时 |
| `_PTY_POLL_INTERVAL` | 0.01 | PTY 空读轮询间隔 |
| `_SNAPSHOT_LIMIT_BYTES` | 128*1024 | 快照上限 |
| `_HEARTBEAT_INTERVAL_DEFAULT` | 30 | 心跳默认值 |

构造函数新增 `heartbeat_interval` 参数，心跳循环使用 `self.heartbeat_interval`。

### 3. `agent/app/transport/agent_message_handler.py` — 内部常量
- `_LARGE_INPUT_LOG_THRESHOLD = 1024` — 终端输入日志阈值

### 4. `agent/app/core/pty_wrapper.py` — 内部常量
| 常量 | 值 | 用途 |
|------|-----|------|
| `_EXEC_SYNC_TIMEOUT` | 5.0 | exec pipe 同步超时 |
| `_READ_BUF_SIZE` | 65536 | PTY 单次读取字节数 |
| `_SIGKILL_WAIT` | 1.0 | SIGKILL 后等待超时 |

### 5. `agent/app/tools/mcp_client.py` — 内部常量
- `_MCP_STOP_TIMEOUT = 5` — MCP Server 停止超时

### 6. `agent/local_server.py` — 内部常量
- `_DISCOVERY_HEALTH_TIMEOUT = 2` — 健康检查超时
- `_DISCOVERY_SCAN_TIMEOUT = 1` — 端口扫描超时

### 7. `agent/app/cli.py` — CLI 参数透传
- `run`/`start`/`configure` 命令新增 `--retry-delay` 和 `--heartbeat-interval` 选项
- WebSocketClient 构造时传入 `retry_delay` 和 `heartbeat_interval`

## 向后兼容

- Config JSON 中 `retry_delay` 和 `heartbeat_interval` 有默认值，旧配置文件无需修改
- CLI `--retry-delay` 和 `--heartbeat-interval` 有默认值，不指定时行为不变
- WebSocketClient 构造函数新参数有默认值，不影响已有调用方
