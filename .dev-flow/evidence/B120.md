## 关键架构决策

### 1. 集中式依赖模块 (`_deps.py`)
所有 store/ws/services 层的函数统一从 `_deps.py` re-export。路由模块使用 `from app.api import _deps` + `_deps.func()` 模式调用。

**原因**: Python `from X import Y` 创建的是局部绑定，`patch("module.attr")` 只修改模块属性不影响已导入模块的局部变量。使用 `import module` + `module.func()` 模式，属性查找在调用时发生，使 `patch("module.func")` 生效。

### 2. `ws_agent.py` 不使用 `_deps`
`ws_agent.py` 直接从 `app.store.session` 导入。测试中针对 broadcast 相关功能直接 mock `app.ws.ws_agent.update_session_terminal_status`，而非通过 `_deps`。

### 3. `agent_session_manager.py` 的延迟导入
`_record_conversation_event` 使用延迟导入 `from app.store.database import append_agent_conversation_event`。测试需同时 mock:
- `app.api._deps.append_agent_conversation_event` (路由层调用)
- `app.store.database.append_agent_conversation_event` (agent loop 内部调用)

### 4. `_check_assistant_plan_rate_limit` 使用模块导入
`assistant_api.py` 使用 `from app.api import assistant_plan_helpers as _plan_helpers` + `_plan_helpers._check_assistant_plan_rate_limit()` 模式，使 mock `app.api.assistant_plan_helpers._check_assistant_plan_rate_limit` 生效。

### 5. 向后兼容重导出
`runtime_api.py` 重导出所有被测试 mock 的符号，确保 `patch("app.api.runtime_api.xxx")` 仍有效。
