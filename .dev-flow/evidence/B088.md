## Execution Evidence

Implemented server-authoritative history reconstruction and terminal close cleanup:

- Added `AgentSession.message_history` and passed it into `terminal_agent.run_agent(message_history=...)`.
- Runtime `run` now rebuilds `message_history` from server `agent_conversation_events` before appending the current `user_intent`.
- Conversation events are converted into Pydantic AI `ModelRequest` / `ModelResponse` messages, covering user intents, answers, assistant questions, traces, results, and errors without relying on client prompt stitching.
- Terminal close now closes the active Agent conversation using `close_agent_conversation`, publishes the `closed` event to active conversation stream subscribers, and cancels the active in-memory Agent session.
- `terminal_closed` messages from the desktop Agent trigger best-effort conversation cleanup.
- Stale/offline Agent cleanup triggers best-effort cleanup for all non-closed terminal conversations in the session.
- Closed conversation fetch returns `410 closed_terminal` and does not return historical events.
- Close cleanup is best-effort so terminal close itself is not blocked by a missing/uninitialized conversation table in older tests or partial deployments.
