## Execution Evidence

### Implementation Summary

实现 HTTP 登录层同端设备并发限制，核心机制为 Redis `token_version:{session_id}:{view_type}`。

**改动文件：**

1. `server/app/auth.py` — 核心变更
   - 新增 `TokenVerificationError` 异常类（继承 HTTPException，携带 error_code）
   - 新增 `normalize_view_type()` — 规范化 view_type，非法值按 mobile 处理
   - 新增 `_token_version_key()` — 统一 Redis key 构造
   - 新增 `get_token_version()` / `increment_token_version()` — Redis 版本读写
   - 修改 `generate_token()` — 新增 token_version + view_type 可选参数
   - 修改 `verify_token()` — 抛出 TokenVerificationError（含 error_code）
   - 新增 `async_verify_token()` — JWT 校验 + Redis 版本比对
   - 修改 `generate_refresh_token()` — 新增 view_type 参数写入 payload

2. `server/app/__init__.py` — 新增 TokenVerificationError 异常 handler

3. `server/app/user_api.py` — API 路由变更
   - `UserLogin` / `UserRegister` 新增 `view` 可选字段
   - `register()` — 递增版本 + 签发带版本 token
   - `login()` — 递增版本 + 签发带版本 token + refresh token 携带 view_type
   - `refresh_token()` — 读取当前版本（不递增）+ 新签发 token 携带版本
   - `get_session_state()` — 使用 `async_verify_token` 替代 `verify_token`

4. `server/tests/test_login_device_limit.py` — 33 个测试
