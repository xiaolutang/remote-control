# F009: 修复刷新终端时选中态乱串

## Task Info
- **Task ID**: F009
- **Module**: client
- **Priority**: high
- **Status**: completed

## Acceptance Criteria
1. `DesktopWorkspaceController.selectedTerminal` getter 为纯读取，不修改 `_selectedTerminalId`
2. `_syncSelectedTerminalId()` 在数据变更点显式调用（`attachRuntimeController`、`_syncDesktopState`）
3. `_resolveSelectedTerminalId` 在终端列表变化时正确回退到第一个非 closed 终端
4. 现有测试全部通过，不受影响

## Implementation

### 根因
`selectedTerminal` getter 内部调用 `_resolveSelectedTerminalId()`，该方法会修改 `_selectedTerminalId`。每次访问 getter（包括 build 方法）都触发副作用，异步刷新 terminals 期间导致选中态跳到第一个终端。

### 修复
- getter 改为纯读：`_findTerminal(runtime.terminals, _selectedTerminalId)`
- 新增 `_syncSelectedTerminalId()` 在数据变更点显式调用
