## Manual Smoke Test Procedures

### Smoke 1: Agent 冷启动 -> 终端列表恢复

**前置条件**: 完整部署环境（Server + Agent + Client），至少有一个已创建的终端

**测试步骤**:
1. 确认 Client 已连接到 Server，终端列表显示正常
2. 停止 Agent 进程（模拟 Agent 冷关机）
3. 等待 30 秒（确保心跳超时）
4. 重新启动 Agent 进程
5. 等待 Agent 重新连接到 Server
6. 在 Client 上刷新设备列表
7. 检查终端列表是否恢复
8. 点击一个终端的"连接"按钮
9. 验证终端内容正确加载

**预期结果**:
- Agent 重新上线后，设备状态从"离线"变为"在线"
- 终端列表恢复正确（与 Agent 关闭前一致）
- 终端可以正常连接并显示内容

### Smoke 2: Agent 启动失败 -> 错误提示 + 重试入口

**前置条件**: 完整部署环境，但 Agent 配置错误或端口被占用

**测试步骤**:
1. 配置 Agent 使用错误的 Server URL 或占用端口
2. 启动 Agent
3. 在 Client 上观察错误提示
4. 修正 Agent 配置
5. 点击"重试"或重新启动 Agent
6. 验证恢复正常

**预期结果**:
- Agent 启动失败时，Client 显示设备"离线"状态
- 错误信息清晰（"设备离线"提示）
- "新建智能终端"按钮被禁用，tooltip 显示不可用
- 修正配置后重试，设备恢复在线，终端功能正常
