## 完成内容

### 1. 新增文件

#### `server/evals/integration.py` — 集成测试框架核心

**IntegrationRunner 类**:
- `build_and_deploy()` — 调用 `deploy/build.sh` 构建 Docker 镜像，`docker compose up -d` 启动
- `wait_for_healthy(timeout)` — 轮询 `/health` 端点，等待服务就绪
- `tear_down()` — `docker compose down` 清理
- 上下文管理器支持（`with IntegrationRunner() as runner:`）
- Docker 构建失败时抛出 `DockerBuildError`（graceful，不 hang/crash）
- 健康检查超时时抛出 `HealthCheckTimeout`

**IntegrationEvalClient 类**:
- `setup()` — 注册测试用户获取 JWT token
- `call_agent_api()` — 通过真实 HTTP API 调用 Agent（SSE run / Assistant Plan）
- `get_or_create_device()` — 获取 Agent 设备 ID
- `list_terminals()` / `create_terminal()` — 终端管理
- `_parse_sse_response()` — SSE 流式响应解析
- `_call_terminal_agent()` — Terminal Agent SSE run API
- `_call_assistant_plan()` — Assistant Plan API（不依赖 Terminal）

**run_integration_task()** — 单 task 集成执行函数，与 unit mode 的 `_execute_single_trial` 对齐

### 2. 修改文件

#### `server/evals/__main__.py` — CLI 增加 --mode 参数

- `run` 子命令增加 `--mode unit|integration`（默认 unit）
- `run` 子命令增加 `--base-url`、`--skip-build`、`--health-timeout` 参数
- `_cmd_run()` 根据 mode 分发到 `_cmd_run_integration()` 或原有 unit 逻辑

- **现有 unit mode 逻辑完全不变**

## 设计决策

1. **integration 模式不依赖 LLM 配置** — 直接打 HTTP API，不需要 EVAL_AGENT_* 环境变量
2. **上下文管理器保证 tear down** — 即使 eval 执行异常，`finally` 块保证 `docker compose down`
3. **多 API 路径支持** — Terminal Agent SSE run + Assistant Plan，适应不同测试场景
4. **SSE 流式解析** — 支持解析 `data: {"type": "result", ...}` 格式的 SSE 流
5. **subprocess 管理而非 Docker SDK** — 复用现有 `deploy/build.sh` 和 `docker compose`，无需额外依赖
6. **httpx 做为 HTTP 客户端** — 项目已有依赖，支持 SSE streaming

## 风险

- **config**: Docker 构建耗时（分钟级），通过 `--skip-build` 可跳过
- **startup**: docker compose 依赖 gateway/infra-network 外部网络，本地测试需确保基础设施已启动
