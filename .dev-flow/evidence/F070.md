### Acceptance Criteria

1. 切换到另一个终端后内容立即显示（不白屏）
2. 切回原终端时 scrollback 内容保留（Terminal 实例被复用）
3. 连续快速切换 3 次不出现白屏
4. WebSocket 重连中显示已有 loading 状态（确认已有逻辑正常工作）
5. 切换到已断开连接的终端显示明确状态而非空白

### Architecture Constraints

- Terminal 实例缓存必须和现有 WebSocket session 缓存使用同一 key，对齐 `deviceId::terminalId`
- auth-failed / logout / disconnectAll 之后必须清空 Terminal 缓存，不能把旧用户 scrollback 泄漏到下一次会话
- 不改 workspace 的 `ValueKey(terminalId)` 结构，问题应收敛在 terminal state 管理层
