### Acceptance Criteria

1. 工作台 Scaffold（主路径）移动端设置 `resizeToAvoidBottomInset: false`
2. `TerminalScreen` 独立 Scaffold（旧路径）移动端也设置 `resizeToAvoidBottomInset: false`
3. 桌面端两个 Scaffold 均保持默认行为
4. 移动端弹起键盘时终端区域不缩小不闪烁
5. 移动端键盘弹起时不发送 resize 消息到远程 PTY
6. 桌面端终端显示不受移动端键盘操作影响
7. 终端输入功能正常（键盘仍可输入文字）
8. 移动端弹起键盘后，底部提示符/光标仍可见（不被键盘永久遮挡）

### Architecture Constraints

- 必须遵守不变量 #35：移动端终端不得将本地 viewport 变化升级为全局 PTY resize
- 修复范围只限两条终端页面入口的 Scaffold keyboard resize 策略
- 不允许为修复键盘 resize 再次引入桌面端行为回归

### Code Review

- 执行方式: local self-review
- 结果: pass
- 结论:
  - 根因在 Scaffold 默认 keyboard inset 行为，不在 WebSocket 或 PTY 层
  - 两条终端入口都已覆盖，避免只修工作台主路径而漏掉旧入口
  - 桌面端显式保持 `true`，避免把“移动端禁 resize”误扩散到桌面
