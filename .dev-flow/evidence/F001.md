### Key Implementation Decisions

1. **createDisabled 代替 maxTerminals**: Widget 不持有业务判断逻辑（terminal 数量 vs 上限）。由调用方基于权威源 `RuntimeDevice.canCreateTerminal` 传入 `createDisabled` 布尔值。这样 widget 不重复实现 `agentOnline && activeTerminals < maxTerminals` 的权威判断。
2. **Widget 通过 terminal.canAttach 控制可切换性**: Closed terminal 仍然渲染为 tab（支持 F004 的上下文菜单场景），但 InkWell.onTap 为 null（不可切换），文字颜色 dimmed（alpha 0.38）。使用 `RuntimeTerminal.canAttach` 权威属性判断。
3. **CompactTabStrip 不使用 SingleChildScrollView**: 避免水平滚动手势与 fling-to-switch 冲突。使用 Expanded 让 tab 自动等分空间。标题使用 `Flexible` + `TextOverflow.ellipsis` 在空间不足时截断。
4. **Tooltip 放在 InkWell 外面**: 避免与 InkWell 的手势竞争。CompactTabStrip 完全不使用 Tooltip（移动端 tab 已直接显示编号+标题）。
5. **CompactTabStrip 使用 GestureDetector.onHorizontalDragEnd**: 基于 velocity 方向判断滑动方向，忽略弱滑动（<200 velocity）。
6. **TerminalTabBar 支持 secondary click**: 通过 `GestureDetector.onSecondaryTapUp` 支持桌面端右键上下文菜单，暴露坐标信息供 PopupMenu 定位。`onContextMenu` 签名为 `void Function(String terminalId, Offset position)?`。
7. **使用 `withValues(alpha: 0.38)` 代替 `withOpacity(0.38)`**: 与项目现有代码收敛一致。
8. **Tooltip 禁用文案使用中性措辞**: `createDisabled` 时显示"新建终端 (不可用)"而非"已达到终端上限"，因为禁用原因可能是离线或达到上限。
