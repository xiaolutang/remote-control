## 问题分析

`WebSocketService.send()` 使用 `_channel!.sink.add(message)` 是 fire-and-forget，send 后消息仅进入发送队列但未真正 flush 到网络。

`_injectAiPrompt()` 在 `service.send()` 后立即执行 `_archiveAgentTurn()` 和 `setState()` 清空状态。

`_executeAgentResult()` 在 `service.send()` 后立即执行 `widget.onClose()` 关闭面板。

如果消息还未真正发送到网络，状态清理就已完成，导致长 prompt 注入可能被截断。

## 修复方案

### 1. WebSocketService.send() 改为返回 Future<void>

在 `sink.add(message)` 后，通过 `IOWebSocketChannel.ready` Future 等待消息 flush 到网络。

```dart
Future<void> send(String data) async {
    // ... 构建 message ...
    _channel!.sink.add(message);
    if (_channel is IOWebSocketChannel) {
      await (_channel as IOWebSocketChannel).ready;
    }
}
```

### 2. _injectAiPrompt() 改为 async，await send 后再归档

```dart
Future<void> _injectAiPrompt() async {
    // ...
    await service.send('${result.aiPrompt}\r');
    // send 完成后再归档和清空状态
    _archiveAgentTurn(result: result);
    // ...
}
```

### 3. _executeAgentResult() await send 后再关闭面板

```dart
Future<void> _executeAgentResult() async {
    // ...
    await service.send(input);
    // send 完成后再关闭面板
    widget.onClose();
}
```

### 4. Mock 类同步更新签名

- `MockWebSocketService.send`: `void` -> `Future<void> async`
- `_FakeWebSocketService.send` (terminal_session_manager_test.dart): 同上

## 兼容性分析

`send` 从 `void` 改为 `Future<void>` 是向后兼容的：
- `terminal_screen.dart:506` (`_sendSpecialKey`): fire-and-forget，忽略返回值，行为不变
- `terminal_screen.dart:1594` (TuiSelector onSelect): 同上
